This guide walks you through installing Kitaru, verifying the CLI, and running
your first small flow.

Kitaru requires **Python 3.11 or newer**.

## Install

<Tabs items={["uv (recommended)", "pip"]}>
  <Tab value="uv (recommended)">
    ```bash
    uv add kitaru
    ```
  </Tab>
  <Tab value="pip">
    ```bash
    pip install kitaru
    ```
  </Tab>
</Tabs>

## Verify the CLI

<Tabs items={["uv project", "pip environment"]}>
  <Tab value="uv project">
    ```bash
    uv run kitaru --version
    uv run kitaru --help
    ```
  </Tab>
  <Tab value="pip environment">
    ```bash
    kitaru --version
    kitaru --help
    ```
  </Tab>
</Tabs>

## Run your first flow

Create a small script named `demo.py`:

```python
from kitaru import checkpoint, flow

@checkpoint
def fetch(topic: str) -> str:
    return f"notes about {topic}"

@flow
def demo(topic: str) -> str:
    return fetch(topic).upper()

handle = demo.run("kitaru")
print("Execution:", handle.exec_id)
print("Result:", handle.wait())
```

Then run it:

<Tabs items={["uv (recommended)", "pip environment"]}>
  <Tab value="uv (recommended)">
    ```bash
    uv run demo.py
    ```
  </Tab>
  <Tab value="pip environment">
    ```bash
    python demo.py
    ```
  </Tab>
</Tabs>

What happens here:

1. `@checkpoint` marks a durable unit of work
2. `@flow` defines the outer execution boundary
3. `demo.run(...)` starts a new execution and returns a `FlowHandle`
4. `handle.wait()` blocks until the execution finishes and returns the flow result

If you want to inspect that execution later, the execution ID you printed is the
same ID used by `KitaruClient` and `kitaru executions ...`.

## Prefer a repo example?

If you would rather run a maintained example from this repository than type out
your own `demo.py`, start here:

```bash
uv sync --extra local
uv run -m examples.basic_flow.first_working_flow
```

For the broader catalog, see [Examples](/getting-started/examples).
