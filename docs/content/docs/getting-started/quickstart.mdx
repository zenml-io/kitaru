This guide walks you through installing Kitaru and verifying that everything
works.

## Install

<Tabs items={["uv (recommended)", "pip"]}>
  <Tab value="uv (recommended)">
    ```bash
    uv add kitaru
    ```
  </Tab>
  <Tab value="pip">
    ```bash
    pip install kitaru
    ```
  </Tab>
</Tabs>

## Verify

Check that the CLI is available:

```bash
kitaru --version
```

You should see the installed version number (e.g. `0.1.0`).

Run `--help` to see available commands:

```bash
kitaru --help
```

## What's Next

Kitaru is still in active development, but you can already use:

- durable execution primitives (`@kitaru.flow`, `@kitaru.checkpoint`)
- structured metadata logging (`kitaru.log()`)
- runtime log-store configuration (`kitaru log-store ...`)

If you want to configure runtime logging now, read
[Configure Runtime Log Storage](/getting-started/log-store).

You can also browse the [CLI Reference](/cli) for command details,
or see the [Changelog](/changelog) for the latest updates.
