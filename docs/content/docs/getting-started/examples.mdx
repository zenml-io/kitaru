Kitaru ships a set of runnable examples in the `examples` package, grouped by
topic:

```bash
uv run examples/<group>/<module_name>.py
```

## Install the right extras

| Goal | Install command |
|---|---|
| Core workflow, execution, replay, configuration, and LLM examples | `uv sync --extra local` |
| PydanticAI adapter example | `uv sync --extra local --extra pydantic-ai` |
| MCP query tools example | `uv sync --extra local --extra mcp` |

## Connection context

Examples use whatever Kitaru connection context is already active.

- If you are just trying Kitaru locally, run them as-is.
- If you already have a deployed Kitaru server and want the examples to use it,
  connect first and verify the active context before running the example.

```bash
uv run kitaru login https://my-server.example.com --api-key <API_KEY>
# add --project <PROJECT> or other login flags if your setup requires them
uv run kitaru status
```

## Browse by goal

If you want to...

- **Start with the smallest runnable flow**, run `examples/basic_flow/first_working_flow.py`
- **Inspect executions after the fact**, run `examples/execution_management/client_execution_management.py`
- **Pause for external approval and continue later**, run `examples/execution_management/wait_and_resume.py`
- **Replay only the suffix that changed**, run `examples/replay/replay_with_overrides.py`
- **Track a model call inside a flow**, run `examples/llm/flow_with_llm.py`
- **Wrap an existing PydanticAI agent**, run `examples/pydantic_ai_agent/pydantic_ai_adapter.py`
- **Explore Kitaru from an MCP client**, run `examples/mcp/mcp_query_tools.py`

## Core workflow basics

| Example | Run | Requires | Demonstrates | Related docs |
|---|---|---|---|---|
| Basic flow | `uv run examples/basic_flow/first_working_flow.py` | `uv sync --extra local` | Smallest end-to-end `@flow` + `@checkpoint` example | [Quickstart](/getting-started/quickstart) |
| Structured logging | `uv run examples/basic_flow/flow_with_logging.py` | `uv sync --extra local` | `kitaru.log()` metadata at flow and checkpoint scope | [Logging and Metadata](/concepts/logging) |

| Configuration | `uv run examples/basic_flow/flow_with_configuration.py` | `uv sync --extra local` | `kitaru.configure()` defaults, overrides, and frozen execution specs | [Configuration](/guides/configuration) |

## Recommended path

If you are learning Kitaru from scratch, this order usually feels best:

1. [Quickstart](/getting-started/quickstart) and `examples/basic_flow/first_working_flow.py`
2. `examples/basic_flow/flow_with_logging.py`
3. [Artifacts](/guides/artifacts) and `examples/basic_flow/flow_with_artifacts.py`
4. [Execution Management](/guides/execution-management) and `examples/execution_management/client_execution_management.py`
5. [Wait and Resume](/guides/wait-and-resume) and `examples/execution_management/wait_and_resume.py`
6. [Replay and Overrides](/guides/replay-and-overrides) and `examples/replay/replay_with_overrides.py`
7. [Tracked LLM Calls](/guides/llm-calls) and `examples/llm/flow_with_llm.py`
8. [PydanticAI Adapter](/guides/pydantic-ai-adapter) and `examples/pydantic_ai_agent/pydantic_ai_adapter.py`
9. [MCP Server](/agent-native/mcp-server) and `examples/mcp/mcp_query_tools.py`

If you prefer the GitHub view, the full repo catalog also lives in
[`examples/README.md`](https://github.com/zenml-io/kitaru/blob/develop/examples/README.md).
