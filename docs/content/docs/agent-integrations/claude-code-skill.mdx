Kitaru provides a Claude Code skill that teaches Claude how to write valid
Kitaru flows, checkpoints, waits, and adapter usage patterns.

## What the skill covers

- `@kitaru.flow` and `@kitaru.checkpoint` roles
- `kitaru.wait()` flow-level guardrails
- `kitaru.log()`, `kitaru.save()`, `kitaru.load()`, `kitaru.llm()`
- `.submit()` / `.result()` checkpoint concurrency
- PydanticAI adapter guidance (`kitaru.adapters.pydantic_ai.wrap(...)`)
- Common mistakes (nested flows, non-serializable checkpoint outputs, etc.)

## Install from the marketplace

The Kitaru skill is published as a Claude Code plugin. Add the marketplace and
install with two commands:

```bash
/plugin marketplace add zenml-io/kitaru
/plugin install kitaru@kitaru
```

Once installed, Claude Code will automatically use the skill when you ask it to
write or refactor Kitaru workflows. You can also invoke it explicitly with
`/kitaru-authoring`.

## Manual installation

If you prefer to copy the skill file directly:

```bash
mkdir -p .claude/skills/kitaru-authoring
curl -fsSL https://raw.githubusercontent.com/zenml-io/kitaru/develop/.claude-plugin/skills/kitaru-authoring/SKILL.md \
  -o .claude/skills/kitaru-authoring/SKILL.md
```

## Good prompts to pair with the skill

- "Refactor this script into one `@kitaru.flow` with explicit checkpoints."
- "Add a flow-level `kitaru.wait()` approval gate before publish."
- "Wrap this PydanticAI call in a Kitaru checkpoint and preserve replay safety."
