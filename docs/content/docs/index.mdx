---
title: What is Kitaru?
description: Durable execution for AI agents
---

Kitaru is a framework for making AI agent workflows **persistent**,
**replayable**, and **observable** — without requiring you to learn a graph DSL
or change your Python control flow.

## Vision

Kitaru provides a small set of primitives — workflows, checkpoints, wait
points, tracked LLM calls, artifacts, and structured logging — that let you
write normal Python code while getting durable execution guarantees underneath.
When something fails halfway through a multi-step agent workflow, Kitaru can
reuse earlier checkpoint results instead of starting over from scratch.

<Callout type="info">
  Kitaru is still early, but the core user-facing surface is already shipped:
  `@flow`, `@checkpoint`, `kitaru.log()`, `kitaru.save()`, `kitaru.load()`,
  `kitaru.configure()`, `kitaru.wait()`, `kitaru.llm()`, `KitaruClient`,
  stack helpers, runtime log retrieval, `kitaru-mcp`, Claude Code skills, and
  the PydanticAI adapter.
</Callout>

## Current Status

The current release includes:

- Core SDK execution primitives: `@flow` and `@checkpoint`
- Structured metadata logging with `kitaru.log()`
- Artifact persistence + cross-execution reads with `kitaru.save()` / `kitaru.load()`
- Durable wait/resume via `kitaru.wait()` plus `input(...)` / `resume(...)`
- Tracked LLM calls via `kitaru.llm()` with model aliases and secret-backed credentials
- Runtime configuration defaults via `kitaru.configure(...)`
- Stack selection + runtime log-store commands (`stack ...`, `log-store ...`)
- Execution management via `KitaruClient` and `kitaru executions ...`
- Runtime log retrieval via SDK, CLI, and MCP `get_execution_logs`
- Secrets and model alias management from the CLI
- Agent-native integrations: MCP server, Claude Code skills, and PydanticAI adapter
- Typed exceptions and structured failure journaling

More adapters and higher-level ergonomics are still under active development.
