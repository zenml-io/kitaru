---
title: What is Kitaru?
description: Durable execution for AI agents
---

Kitaru is a framework for making AI agent workflows **persistent**,
**replayable**, and **observable** — without requiring you to learn a graph DSL
or change your Python control flow.

## Vision

Kitaru will provide a small set of primitives — sagas, checkpoints, wait
points, and structured logging — that let you write normal Python code while
getting durable execution guarantees underneath. When something fails halfway
through a multi-step agent workflow, Kitaru replays from the last checkpoint
instead of starting over.

<Callout type="info">
  Kitaru is in early development. The package is installable and provides a CLI,
  but the SDK primitives are not yet available. This documentation covers what
  you can use today.
</Callout>

## Current Status

Kitaru **0.1.0** includes:

- The `kitaru` CLI with `--help` and `--version`
- Project infrastructure (CI, testing, linting, type checking)

The SDK primitives and framework adapters are under active development.
