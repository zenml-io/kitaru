Add or update documentation alongside code changes.

- Add docstrings to any public functions you create or modify
- Follow the project's existing docstring style (Google, NumPy, or reST)
- Update README sections if user-facing behavior changes
- Keep docstrings concise — explain *why*, not *what*
- Document parameters, return values, and raised exceptions
- If the project has a `docs/` directory, check whether your changes
  affect any published documentation
